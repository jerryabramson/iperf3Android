## 3. iOS Codebase Reviewed — tvOS Support Added

**Goal**: Review the tvOS changes Jerry made to the iOS app.

**Findings from `CLAUDEAI-TVOS-SUPPORT.md`:**

### 7 Root Causes of tvOS Build Failure (All Fixed)
1. `iperf3lib.xcodeproj` had zero tvOS configuration
2. `libiperf-device.a` was iOS-only binary, not tvOS-compatible
3. App target had contradictory SDK settings (`SDKROOT = iphoneos` on tvOS target)
4. Library search path resolved to non-existent `Debug-appletvos` directory
5. Invalid `ADDITIONAL_SDKS[arch=*] = tv` setting
6. iOS-only `Info.plist` keys on tvOS target (launch screens, interface orientation)
7. Dangling `PBXTargetDependency` with no target or proxy

### Solution: XCFramework Approach
- Cross-compiled iperf3 for 4 platform slices:
  - `libiperf-iphoneos.a` (iOS device, arm64)
  - `libiperf-iphonesimulator.a` (iOS simulator, arm64+x86_64)
  - `libiperf-appletvos.a` (tvOS device, arm64)
  - `libiperf-appletvsimulator.a` (tvOS simulator, arm64+x86_64)
- Bundled into `libiperf3.xcframework`

### tvOS UI Work
- `ContentView.swift` split into `#if os(tvOS)` / `#else` branches
- `TVRadioGroup` — custom Button-based radio rows (since `Picker` has no persistent selection indicator on tvOS)
- `#if os(iOS)` guards for `UIKeyboardType`, `UIColor.systemGray*`, `UIColor.systemBackground`
- Two-panel tvOS layout: `tvControlPanel` + `tvOutputPanel`
- Deployment targets raised to **17.0** (both iOS and tvOS) for `onChange(of:initial:_:)`

### Rebuild Notes
- Configure cache override required: `ac_cv_header_endian_h=no ac_cv_header_sys_endian_h=no`
- Use `make -C src libiperf.la` (not `make all`) — `daemon()` unavailable on tvOS
